Game Engine

January 2018 - Now
position: engine engineer

overview

A 3D game engine I am working on, mainly about real-time rendering. The engine is developed by C++, Direct3D 11, Lua, and HLSL.

The development has two parts, which are responding to classes in three semesters. Some of the write-ups can be checked in my blog. The first part is to build the structure and do some low level stuff such as memory pool, smart pointersï¼Œ and cache-friendly optimization. Also, some basic features like rigidbody, collision, math library, and delegated are added.

The Second part is to make the platform-independent graphics library. I developed all the classes needed and responding files and asset builders. Used some tricks like sorting and cache to optimize the render pipeline. Meanwhile, I implemented some lighting models in shaders.

What I Did

  • Implemented render library by DirectX, optimized rendering pipeline. Implemented PBR shaders.
  • Wrote asset building Lua scripts. Customized readable Lua asset files and built them into binary files.
  • Implemented memory pools, allocators, and smart pointers to optimize memory allocation.
  • Implemented collision detection and a physical system, used SSE to optimize matrix calculation.
